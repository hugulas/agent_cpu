# Chapter Expansion Checklist

Use this checklist when expanding one chapter of a long review.

## Required outcomes

- The chapter explains `why`, not only `what`
- Each major claim has a nearby citation
- Figures support arguments rather than decorate the chapter
- The chapter can stand on its own if read separately

## Expansion pattern

For each major subsection:

1. Start with the problem or claim
2. Explain the mechanism
3. Add evidence, numbers, or figure references
4. State the implication

## Common fixes

### If the writing is too compressed

- Split one dense paragraph into:
  - observation
  - mechanism
  - implication

### If the chapter is under-evidenced

- Add a compact evidence table
- Add one extracted paper figure or system diagram
- Attach citations to the sentence that makes the causal claim

### If the chapter has too many claims

- Group them into 2-4 subarguments
- Add one transition sentence that explains how they connect

## Figure guidance

Prefer figures that do one of these jobs:

- show system topology
- show slowdown/bottleneck evidence
- show memory hierarchy or state flow
- show algorithmic mechanism

Avoid adding a figure if you cannot explain in one sentence why it matters.
