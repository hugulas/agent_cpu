# 评审意见：11-16 子章节共性问题与主线三、四整体评价

## 主线三整体评价（10-13）

主线三（MoE 为什么会把 host-side orchestration 推到前台）的展开总体成功：
- **10** 定义问题（稀疏计算优势不自动转化）
- **11** 展开三条解法路线（FluxMoE / FineMoE / SpecMoEOff）
- **12** 深化控制面问题（expert skew 与动态平衡）
- **13** 收束到工业吸收状态（问题域已吸收，解法未全面吸收）

递进逻辑清晰，且能持续锚定"CPU 角色"这一核心命题。最大问题是 12/13 的章节顺序颠倒和孤儿内容未清理。

## 主线四整体评价（14-16）

主线四（PD 分离与跨池控制平面）的展开同样成功：
- **14** 建立 agentic workload 与 PD 分离的亲和性（三个特征）
- **15** 展开从单集群 PD 到 PraaS 的升级路径
- **16** 收束到 CPU 的直接设计要求和平台信号

14 的论证有具体数字支撑（`100` sub-agents / `1,500+` tool calls / `4.5x`），15 有完整的定量证据链（`+54%` / `-64%` / `+32%`），16 能收束到 Vera / Grace 平台信号。最大问题是 14 的严重结构混乱和 15/16 的孤儿内容。
